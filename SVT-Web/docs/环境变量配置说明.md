# SVT前端环境变量配置说明

## 概述
SVT前端应用使用环境变量来管理不同环境下的配置信息，提高应用的灵活性和可维护性。

## 配置文件
- `env.example` - 环境变量示例文件，包含所有可配置项
- `.env.development` - 开发环境配置文件
- `.env.uat` - UAT测试环境配置文件
- `.env.production` - 生产环境配置文件
- `src/config/env.ts` - 环境变量管理模块

## 使用方法
1. **开发环境**: 使用 `.env.development` 文件（已创建）
2. **UAT测试环境**: 使用 `.env.uat` 文件（已创建）
3. **生产环境**: 根据 `.env.production` 示例文件修改配置
4. **自定义环境**: 可以创建 `.env.local` 文件覆盖默认配置
5. 重启开发服务器使配置生效

## 环境文件优先级
Vite 会按以下优先级加载环境文件：
1. `.env.local` (本地覆盖，所有环境)
2. `.env.[mode].local` (本地覆盖，特定环境)
3. `.env.[mode]` (特定环境)
4. `.env` (默认)

## 配置项说明

### 应用基本信息
| 变量名 | 默认值 | 说明 |

| `VITE_APP_TITLE` | SVT 管理系统 | 应用标题，显示在登录页面 |

## 环境变量使用示例

### 在组件中使用
```typescript
import { appConfig, getAdminContactText, isDevelopment, isProduction } from '@/config/env';

// 使用应用标题
const title = appConfig.appTitle;

// 使用管理员联系信息
const contactText = getAdminContactText();

// 环境判断
if (isDevelopment) {
  console.log('当前为开发环境');
}

// 使用API配置
const apiUrl = `${appConfig.apiBaseUrl}/users`;
```

### 在Vite配置中使用
```typescript
import { loadEnv } from 'vite';

export default defineConfig(({ mode }) => {
  const env = loadEnv(mode, process.cwd(), '');
  const apiBaseUrl = env.VITE_API_BASE_URL || 'http://localhost:8080';
  
  return {
    // 配置...
  };
});
```

## 部署注意事项

### 开发环境 (Development)
- 使用默认配置即可
- 确保后端服务运行在 `http://localhost:8080`
- 开启调试模式方便开发
- AES加密默认关闭

### UAT测试环境 (UAT)
- 修改 `VITE_API_BASE_URL` 为UAT环境后端地址
- 设置 `VITE_APP_ENV=uat`
- 关闭调试模式 `VITE_ENABLE_DEBUG=false`
- 开启AES加密 `VITE_AES_ENABLED=true`
- API超时时间适当增加至15秒

### 生产环境 (Production)
- 修改 `VITE_API_BASE_URL` 为生产环境后端地址
- 设置 `VITE_APP_ENV=production`
- 关闭调试模式 `VITE_ENABLE_DEBUG=false`
- 开启AES加密 `VITE_AES_ENABLED=true`
- API超时时间增加至20秒
- 配置正确的管理员联系信息

### 环境配置示例

#### UAT环境配置
```env
# .env.uat
VITE_APP_TITLE=SVT 管理系统 (UAT)
VITE_APP_ENV=uat
VITE_API_BASE_URL=http://uat.svt.com/api
VITE_API_TIMEOUT=15000
VITE_ADMIN_EMAIL=admin@svt.com
VITE_ENABLE_DEBUG=false
VITE_AES_ENABLED=true
```

#### 生产环境配置
```env
# .env.production
VITE_APP_ENV=production
VITE_API_BASE_URL=https://prod.svt.com/api
VITE_API_TIMEOUT=20000
VITE_ADMIN_EMAIL=admin@yourcompany.com
VITE_ADMIN_PHONE=400-123-4567
VITE_ENABLE_DEBUG=false
VITE_AES_ENABLED=true
```

## 安全注意事项
- 不要在环境变量中存储敏感信息（如密钥、密码）
- 生产环境的 `.env` 文件不应提交到版本控制系统
- 确保 `.env` 文件已添加到 `.gitignore`
- AES加密密钥应在各环境中使用不同的值
- 建议在生产环境中使用环境特定的密钥管理服务

## 环境变量管理最佳实践

### 开发阶段
1. 使用默认的 `.env.development` 配置
2. 如需自定义，创建 `.env.local` 覆盖特定配置
3. 利用 `VITE_ENABLE_DEBUG=true` 进行调试

### 测试阶段
1. 使用 `.env.uat` 配置进行集成测试
2. 开启AES加密进行安全测试
3. 验证与后端的集成性

### 生产部署
1. 使用 `.env.production` 配置
2. 严格遵循安全规范
3. 定期检查和更新配置

## 常见问题解答

### Q: 为什么环境变量修改后没有生效？
A: 需要重启开发服务器（`npm run dev`）才能加载新的环境变量。

### Q: 怎么在代码中使用环境变量？
A: 推荐使用 `src/config/env.ts` 中定义的 `appConfig` 对象，这样可以获得类型安全和默认值支持。

### Q: 可以创建自定义环境吗？
A: 可以。创建 `.env.[mode]` 文件，然后使用 `npm run dev -- --mode [mode]` 启动。

### Q: AES加密密钥如何生成？
A: 可以使用在线工具或Node.js生成随机的32字节密钥，然后转换为Base64格式。
